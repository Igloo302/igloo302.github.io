Git网络加速|为Git设置代理

Written by Igloo with ♥ on February 25, 2020 in

Git使用，尤其是Clone Github的respository的时候往往慢得令人发指，本文使用了2种方法提高速度。

设置代理

直接使用SS和V2ray的全局模式并不起作用，需要通过以下方法设置代理。

  1. 打开代理软件，查看本地HTTP监听端口或SOCKS端口
  2. 如果设置全局代理，则所有Git的操作都经过代理：
# HTTP监听端口为1080
git config --global http.proxy http://127.0.0.1:1080
git config --global https.proxy https://127.0.0.1:1080

# Socks5端口为1086
git config --global http.proxy socks5://127.0.0.1:1086
git config --global http.proxy socks5://127.0.0.1:1086

# 取消全局代理
git config --global --unset http.proxy
git config --global --unset https.proxy
  1. 如果只对GitHub设置代理：
# HTTP监听端口为1080
git config --global http.https://github.com.proxy https://127.0.0.1:1080
git config --global https.https://github.com.proxy https://127.0.0.1:1080

# Socks5端口为1086
git config --global http.https://github.com.proxy socks5://127.0.0.1:1086
git config --global https.https://github.com.proxy socks5://127.0.0.1:1086

# 取消Github代理
git config --global --unset http.https://github.com.proxy
git config --global --unset https.https://github.com.proxy
  1. 此代理方法只对HTTP(S)协议有效，无法用于SSH协议，也就是说只能使用git clone https://www.github.com/xxxx/xxxx.git

修改Host

查询DNS

使用命令nslookup github.global.ssl.fastly.Net和nslookup github.com查询域名对应的IP

PS C:\Users\>  nslookup github.global.ssl.fastly.Net
服务器:  UnKnown
Address:  192.168.6.2

非权威应答:
名称:    github.global.ssl.fastly.Net
Address:  151.101.129.194

PS C:\Users\>  nslookup github.com
服务器:  UnKnown
Address:  192.168.6.2

非权威应答:
名称:    github.com
Address:  13.229.188.59

修改Host

打开host文件，Windows的Host文件在 C:\Windows\System32\drivers\etc\hosts中，Mac和Linux在在/etc/hosts中。在末尾添加两行

151.101.129.194 github.global.ssl.fastly.net
13.229.188.59 github.com

刷新DNS缓存

Linux：

sudo /etc/init.d/networking restart

Windows：

ipconfig /flushdns

macOS：

sudo killall -HUP mDNSResponder
